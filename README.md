## Project 1 and 2

- Project 1 and 2 cover the installation of ONOS and Mininet, the setup of the experimental environment used in later projects, and basic observations of the OpenFlow protocol and Flow Rule installation. Since these topics are foundational, they are not included in detail in this repository.

## Project 3: SDN-enabled Learning Bridge (SDN)

- An SDN-enabled Learning Bridge shifts the MAC address learning and packet forwarding decisions from traditional switches to a centralized SDN controller, while the switch is responsible only for forwarding packets according to the flow rules installed by the controller.

## Project 4: Unicast DHCP Application (SDN)

- A Unicast DHCP Application is an implementation or application that uses the Dynamic Host Configuration Protocol (DCHP) to dynamically assign IP addresses to hosts, while using unicast transmission during part or all of the DHCP communication process.

## Project 5: Proxy ARP (SDN)

- In this project, the SDN controller acts as a Proxy ARP entity, responding to ARP requests on behalf of destination hosts and enabling communication without requiring direct ARP resolution between hosts.

## Project 6: Software Router and Containerization (NFV)

- Software Router and Containerization refers to deploying a software-based router inside containers, enabling flexible deployment, efficient resource utilization, and scalable network services in modern cloud-native and SDN environments.

## Final Project: SDN Network as Virtual Router (SDN and NFV)

- SDN Network as Virtual Router is an SDN architecture in which the entire network operates as a single virtual router, with the SDN controller making centralized routing decisions and the switches forwarding packets based on the installed flow rules.
